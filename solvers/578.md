# Project Euler 578 — Solution Summary

We count **decreasing prime power** integers `n ≤ N`:

If

`n = p1^a1 p2^a2 … pk^ak` with `p1 < p2 < … < pk`,

then we require:

`a1 ≥ a2 ≥ … ≥ ak`

(and `n = 1` counts).

Target: `C(10^13)`.

---

## Main techniques

### 1) Recursive construction enforcing exponent monotonicity
We build the factorization in increasing prime order.

At a recursion state `(limit, start_prime_index, max_exp)`:

- We may append a prime power `p^e` where `2 ≤ e ≤ max_exp`.
- After choosing `e`, the next exponents must be ≤ `e`, so we recurse with `max_exp := e`.

Crucially, **once we ever choose exponent `1`**, all later exponents must also be `1`.
So the “rest” of any branch with `max_exp = 1` is simply a **squarefree product** of larger primes.

---

### 2) Counting squarefree tails with Möbius (no π(x))
The number of squarefree integers ≤ x is:

\[
Q(x)=\sum_{i=1}^{\lfloor\sqrt{x}\rfloor}\mu(i)\left\lfloor\frac{x}{i^2}\right\rfloor
\]

We compute Möbius `μ(i)` up to `⌊√N⌋` via a linear sieve, then compute `Q(x)` efficiently by
grouping ranges where `⌊x/i^2⌋` is constant using prefix sums of μ.

This avoids prime counting entirely, even though primes above the sieve bound contribute.

---

### 3) Squarefree tail with minimum prime constraint
We need squarefree numbers whose primes are all **greater than the last chosen prime**.

For small excluded-prefix sizes, we compute:

`count = Q(x) - (numbers whose smallest prime factor is among excluded primes)`

Each squarefree number has a unique smallest prime factor `p`,
and those with smallest prime `p` are exactly:

`p * m` where `m` is squarefree and all prime factors are greater than `p`.

For large `start_idx`, the remaining `x` is small (because a large prime power was used),
so we count by explicit DFS enumeration.

---

### 4) Heavy memoization
We memoize:

- squarefree tail counts `(x, start_idx)`
- recursion states `(limit, start_idx, max_exp)`

This keeps the runtime practical for `N = 10^13`.
