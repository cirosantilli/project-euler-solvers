# Maximal Perimeter (Problem 562)

## Key ideas
- An empty lattice triangle has area `1/2` (Pick's theorem), so after translation we can write the vertices as `A`, `A+u`, `A+s` with `det(u, s)=1`, and the third edge is `w=s-u`.
- For a fixed `s=(a,b)` with `gcd(a,b)=1`, the unique lattice pair `(u,w)` with `u+w=s` and `det(u,w)=1` is obtained via the modular inverse; this gives the only empty triangle with side vectors `(u,w,s)` up to translation.
- The perimeter is `|u|+|w|+|s|`. For any `s`, the sum `|u|+|w|` is bounded by `sqrt(|s|^2+2)`, so larger `|s|` always gives a larger possible perimeter. This makes it sufficient to check lattice points on the circle boundary (maximal `|s|` for each `x`).
- A triangle is feasible if there exists an integer translation `t` such that `t`, `t+u`, and `t+s` all lie inside the radius‑`r` circle. The intersection of the first two disks is a lens centered at `-s/2`, so it is enough to check integer points in a small square around that center.

## Algorithm
1. Let `R=2r`. For each `x` from `1` to `R`, compute the largest `y` with `x^2+y^2<=R^2`.
2. Skip non-primitive pairs (`gcd(x,y) != 1`).
3. Compute parents `(u,w)` via modular inverse, perimeter `|u|+|w|+|s|`, and only keep candidates that can be translated into the circle.
4. Track the maximum perimeter candidate and compute `R = |u||w||s|/2`, then `T(r)=R/r`.

## Correctness notes
- Checking only the boundary points does not miss the maximum perimeter triangle because any interior point has strictly smaller `|s|`, and the best possible perimeter is monotone in `|s|`.
- The translation test is exact: it scans every integer point in the lens implied by the two diameter endpoints, then checks the third vertex.

## Complexity
- The scan is `O(R)` lattice points with cheap integer work; only a small number of candidates pass the perimeter test and require translation checks.
