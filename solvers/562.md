# Maximal Perimeter — Approach

## Key observations
- An empty lattice triangle has area 1/2 (Pick's theorem), so its edge vectors form a unimodular pair with determinant 1.
- Such a triangle can be written as `A`, `A+u`, `A+u+w` where `det(u,w)=1`.
- For maximal perimeter inside a circle of radius `r`, the triangle is nearly degenerate: `u` and `w` are large and almost parallel, and the long side `s = u + w` is close to the diameter `2r`.
- For any primitive `s = (a,b)` with `a>0`, there is a unique parent decomposition `s = u + w` with `det(u,w)=1`, obtained via a modular inverse.

## Algorithm
1. Let `R = 2r`. Scan integer `x` from `1` to `R` and set `y = floor(sqrt(R^2 - x^2))` (points on or near the circle of radius `2r`).
2. For each `x`, decrement `y` until a primitive `s = (x,y)` is found (at most a few tries in practice).
3. Compute the parents `u,w` of `s` using `p = b^{-1} mod a`, then `u = (p, (pb-1)/a)` and `w = (a-p, b-q)`.
4. Check feasibility by testing a 3x3 grid of candidate translations around `-s/2`. If any translation places all three vertices inside the radius-`r` circle, the candidate is valid.
5. Track the maximum perimeter and finally compute `R = abc/2`, so `T(r) = R/r`.

## Complexity
- The scan is `O(r)` with small constant work per `x`.
- For `r = 10^7`, this runs in under a couple of minutes in PyPy.

## Notes
- The implementation asserts the provided reference values for `T(10)` and `T(100)` within tolerance.
