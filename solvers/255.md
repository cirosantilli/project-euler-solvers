# Project Euler 255 — Rounded Square Roots

This problem defines an **integer-only** version of Heron’s/Newton’s method for
computing a rounded square root.

## Core techniques used

### 1) Integer Newton/Heron iteration
The update step is:

\[
x_{k+1} = \left\lfloor \frac{x_k + \lceil n/x_k \rceil}{2} \right\rfloor
\]

In code, `ceil(n / x)` is implemented safely using integer arithmetic:

* `ceil_div(n, x) = (n + x - 1) // x`

This avoids floating-point error entirely.

### 2) Digit-count initial guess
The starting value `x0` depends only on the number of decimal digits `d` of `n`:

* If `d` is odd: `x0 = 2 * 10^((d-1)/2)`
* If `d` is even: `x0 = 7 * 10^((d-2)/2)`

This is exactly the initialization given in the statement.

### 3) Loop until convergence
The iteration halts once it reaches a fixed point (`x_{k+1} == x_k`). The final
fixed point is the **rounded square root** (no ties occur for integer `n`).

### 4) Verification with the statement’s sample values
`main.py` contains asserts for the statement’s check cases:

* `n = 4321` converges to `66` in `2` iterations.
* The average iterations over all 5-digit integers `10000..99999` is
  `3.2102888889`.

## Output

Running `python main.py` prints the requested 14-digit average, rounded to 10
decimal places:

```
4.4474011180
```
