# Project Euler 386 — Maximum length of an antichain

This repository contains a standalone Python solution (`main.py`) for Project Euler problem 386.

## Key ideas

### 1) Divisors as a ranked poset
If

- \(n = \prod p_i^{a_i}\)

then every divisor corresponds to an exponent vector \((e_i)\) with \(0 \le e_i \le a_i\), and

- \(d_1 \mid d_2 \iff e^{(1)}_i \le e^{(2)}_i\) for all \(i\).

So the divisor poset is a product of chains, and it is **ranked** by:

- \(\Omega(d) = \sum_i e_i\) (number of prime factors of \(d\), counted with multiplicity).

The number of divisors of rank \(k\) is the coefficient of \(x^k\) in:

\[
\prod_i (1 + x + x^2 + \dots + x^{a_i}).
\]

### 2) Maximum antichain = largest rank level
In a ranked poset, a largest antichain is achieved by taking the largest rank level (Sperner property holds for products of chains).

The polynomial above is **palindromic**, so its coefficients are symmetric and peak in the middle. Therefore:

- \(N(n)\) is the coefficient of \(x^{\lfloor \Omega(n)/2 \rfloor}\) in \(\prod_i (1 + x + \dots + x^{a_i})\).

In other words:

- \(N(n)\) is the number of divisors \(d \mid n\) with \(\Omega(d) = \lfloor \Omega(n)/2 \rfloor\).

`main.py` computes this coefficient using a tiny dynamic program (degree ≤ 26 for \(n \le 10^8\)).

### 3) Don’t iterate over all \(n \le 10^8\)
Instead of factoring every integer up to \(10^8\), we:

1. Enumerate all possible **exponent sequences** \((e_1, e_2, \dots, e_k)\) that can occur for \(n \le 10^8\) in increasing-prime order.
   - Here \(k \le 8\) because \(2\cdot3\cdot5\cdot7\cdot11\cdot13\cdot17\cdot19 < 10^8\) but multiplying by 23 exceeds it.
2. For each exponent sequence, count how many integers \(n \le \text{limit}\) have that exact pattern of exponents.
3. Multiply that count by the corresponding \(N\)-value (which depends only on the multiset of exponents).

### 4) Counting factorizations with recursion + strong bounds
To count numbers of the form:

\[
n = p_1^{e_1} p_2^{e_2}\dots p_k^{e_k}, \quad p_1 < p_2 < \dots < p_k,
\]

we recurse over primes. A crucial pruning bound is:

- if there are exponents \(e_i + e_{i+1} + \dots + e_k = S\) remaining, then the next prime \(p\) must satisfy \(p^S \le \text{remaining limit}\),
  because the remaining primes are all ≥ \(p\).

This keeps prime loops small (≤ \(10^4\) for \(10^8\)).

At the final recursion level, we only need “how many primes are in \((p_{\text{prev}}, p_{\max}]\)”, i.e. \(\pi(p_{\max}) - \pi(p_{\text{prev}})\).

### 5) Fast prime counting with Lehmer’s algorithm
To support many \(\pi(x)\) queries up to \(10^8\) efficiently, the solution implements **Lehmer prime counting**:

- uses a small sieve up to 2,000,000,
- uses a cached \(\varphi(x,a)\) (phi) function accelerated by a small primorial wheel (mod 30030),
- memoizes computed \(\pi(x)\) values.

This avoids building a full sieve up to \(10^8\) while keeping \(\pi(x)\) fast.

## Output
Running `main.py` with no input prints the Project Euler answer for \(\sum_{n\le 10^8} N(n)\):

- `528755790`
