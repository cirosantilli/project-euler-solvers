# Project Euler 386 — Maximum Length of an Antichain

This repository contains a fast, pure-Python solution for Project Euler 386.

## Key ideas

### 1) Divisors form a product-of-chains poset
If

- \(n = \prod p_i^{a_i}\),

then every divisor corresponds to a vector \((e_i)\) with \(0 \le e_i \le a_i\), and divisibility is exactly
componentwise order:

- \((e_i) \le (f_i)\) iff \(e_i \le f_i\) for all \(i\).

So the divisor poset is a **Cartesian product of chains**.

### 2) Longest antichain = largest rank size
This poset is naturally *ranked* by

- \(\mathrm{rank}(e) = \sum e_i\).

The number of divisors at rank \(r\) is the coefficient of \(x^r\) in:

\[
\prod_i (1 + x + x^2 + \dots + x^{a_i})
\]

Therefore:

- \(N(n)\) = **maximum coefficient** of that polynomial.

For \(n \le 10^8\), the total degree \(\sum a_i \le 26\), so coefficients are tiny and computed via a small DP.

### 3) Count factorizations by exponent sequences, not by iterating \(n\)
Every \(n>1\) has a unique prime factorization with increasing primes:

\[
n = p_1^{a_1} p_2^{a_2} \dots p_k^{a_k}\quad (p_1 < p_2 < \dots < p_k)
\]

We enumerate all feasible ordered exponent sequences \((a_1,\dots,a_k)\) such that the *smallest* primes
\(2,3,5,\dots\) with those exponents still fit under the limit — this produces only a few thousand sequences.

For each exponent sequence we count how many increasing prime tuples satisfy:

\[
\prod p_i^{a_i} \le 10^8
\]

and multiply by the corresponding \(N(n)\).

### 4) Tight pruning bound keeps prime iteration tiny
When choosing \(p_{\text{pos}}\), later primes are \(\ge p_{\text{pos}}\), so a necessary condition is:

\[
p_{\text{pos}}^{\,\sum_{i\ge pos} a_i} \le \text{remaining}
\]

This dramatically reduces the range of primes that must be *iterated* (it never exceeds \(\sqrt{10^8}=10^4\)).

### 5) Lehmer prime counting π(x)
To count choices for the **last** prime in a tuple efficiently, we need \(\pi(x)\) for \(x\) up to \(10^8\).

The solution uses **Lehmer's prime counting algorithm**, which requires only a sieve up to \(x^{2/3}\)
(\(\approx 2.2 \times 10^5\) for this problem). This avoids generating all primes up to \(10^8\).

## Files

- `main.py` — computes \(\sum_{1\le n \le 10^8} N(n)\) and prints the answer.
- Includes a few small self-tests (brute-force validated offline) as `assert` statements.
