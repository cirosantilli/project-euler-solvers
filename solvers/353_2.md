# Risky Moon

## Overview
We need the minimum summed risk from the north pole to the south pole on a sphere of radius `r`, where stations are all integer lattice points on the sphere. Each road cost depends only on the central angle between consecutive stations.

Key ideas:
- **Integer points via sum of two squares**: For each latitude `z`, points satisfy `x^2 + y^2 = r^2 - z^2`. We generate all `(x, y)` solutions with a Gaussian-integer factorization method, avoiding `O(r^2)` scans.
- **Nearest-neighbor graph**: The optimal path uses only short edges. We build a graph where each point connects to its `k` nearest neighbors by Euclidean distance on the sphere (equivalently, smallest angles).
- **Correctness by saturation**: If the `k`-th neighbor of every point is farther (larger angle) than the best path’s maximum-allowed angle, then all edges that could appear in a better path are included. We double `k` until this condition is met.
- **Dijkstra for shortest path**: With the neighbor graph, we run Dijkstra using edge weight `(arccos(dot/r^2)/pi)^2`.

## Files
- `main.py`: computes `sum_{n=1}^{15} M(2^n - 1)` and prints the 10-decimal answer. Includes the provided test assertion for `M(7)`.

## Complexity
- Point generation is near-linear in the number of lattice points using factorization.
- The graph has `O(k * N)` edges; Dijkstra runs in `O(k * N log N)`.
- `k` is increased only if needed for correctness.
