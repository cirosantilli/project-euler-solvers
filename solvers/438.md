# Project Euler 438 — Solution Summary

This solution enumerates all integer monic polynomials

\[
x^n + a_1x^{n-1} + \cdots + a_n
\]

whose **real roots** \(x_1 < \cdots < x_n\) satisfy:

- all roots are real
- \(\lfloor x_i \rfloor = i\) for \(1 \le i \le n\)

The problem asks for:

\[
\sum_{t} S(t), \quad S(t)=\sum_{i=1}^n |a_i|
\]

for \(n=7\).

---

## Core Techniques Used

### 1) Shift + Integer-value constraints
Define:

\[
g(x) = P(x+1)
\]

Then the constraints on floors mean the roots of \(P\) lie in disjoint unit intervals,
which implies **tight bounds** on the integer values:

\[
P(1), P(2), \dots, P(n+1)
\]

Because:

\[
P(k)=\prod_{i=1}^n (k-x_i)
\]

each factor lies in a short interval, giving a strong bound on \(|P(k)|\) and its sign.

---

### 2) Falling factorial basis (Newton series)
The key simplification is expressing \(g\) in the falling factorial basis:

\[
g(x)=\sum_{k=0}^n d_k \, x^{\underline{k}}
\]

where:

\[
x^{\underline{k}} = x(x-1)\cdots(x-k+1)
\]

Properties:

- \(g\) has integer coefficients **iff** all \(d_k\) are integers
- the forward differences satisfy:

\[
\Delta^k g(0)=k!\,d_k
\]

Monic means:

\[
d_n = 1
\]

So we only need to search integer \(d_0,\dots,d_{n-1}\).

---

### 3) Higher forward-difference inequalities for pruning
Forward differences of order \(r\ge 2\) eliminate the unknown linear terms \(d_0,d_1\),
yielding constraints involving only \(d_2,\dots,d_{n-1}\) plus a constant from \(d_n=1\).

For each order \(r\) and position \(x\):

\[
\Delta^r g(x)=\sum_{k=r}^n d_k\cdot k^{\underline r}\cdot x^{\underline{k-r}}
\]

We compute **necessary bounds** on \(\Delta^r g(x)\) from the previously derived bounds on \(g(0..n)\),
turning the search into solving a small integer feasibility problem.

This reduces the search space from astronomically large to manageable.

---

### 4) Solve \(d_1\) and \(d_0\) via interval intersections
Once \(d_2..d_{n-1}\) are fixed:

- First differences constrain \(d_1\)
- Value bounds constrain \(d_0\)

Each step becomes intersecting integer intervals.

---

### 5) Sturm sequence for exact root counting
Even after all necessary constraints, a candidate polynomial must be verified.

We use a **Sturm sequence** to count real roots in intervals:

- For each \([i,i+1)\) verify exactly **one root**
- Detect and handle possible roots on integer boundaries correctly

This is exact and avoids floating point errors.
