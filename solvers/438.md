# Project Euler 438 — Solution Summary

This solution computes the sum of `S(t)` over all integer coefficient tuples
producing polynomials whose roots satisfy:

- All roots are real
- If the roots are sorted `x1 ≤ x2 ≤ … ≤ xn`, then `floor(xi) = i`

---

## Key Techniques Used

### ✅ 1. Tight bounds on integer values `p(k)`

If a polynomial has one root in each interval `[i, i+1)`, then for integer points
`k = 1..n+1` we have a strict constraint on the magnitude:

\[
|p(k)| = \prod_{i=1}^n |k - x_i|
\le (k-1)!\cdot(n-k+1)!
\]

This gives extremely small ranges for `p(k)` even though the coefficients can be large.

---

### ✅ 2. Forward differences & Newton / falling-factorial basis

Every polynomial can be represented in a falling factorial basis:

\[
p(x)=\sum_{j=0}^{n} c_j (x-1)(x-2)\cdots(x-j)
\]

The coefficients relate to forward differences:

\[
b_j = \Delta^j p(1),\qquad c_j = \frac{b_j}{j!}
\]

For integer power-basis coefficients, the divisibility constraint holds:

\[
b_j \equiv 0 \pmod{j!}
\]

This becomes a strong pruning rule while enumerating valid `p(1..n)` values.

---

### ✅ 3. Monic constraint via nth difference

The polynomial is **monic**, so the `n`th forward difference is fixed:

\[
\Delta^n p(1)=n!
\]

This gives a direct formula for `p(n+1)` from `p(1..n)` and prevents needing to enumerate it.

---

### ✅ 4. Sturm sequence root counting (exact, no floating point)

Even after pruning by bounds and divisibility, some candidates remain that do not
actually place roots in the required `[i,i+1)` intervals.

So every candidate polynomial is checked using **Sturm’s theorem**, which counts
how many real roots fall inside an interval using sign-variation counts of a Sturm chain.

This is done exactly with rational arithmetic (`fractions.Fraction`).

Intervals are validated as:

- exactly one root in `[i, i+1)` for every `i=1..n`
- polynomial must be square-free (no repeated roots)
