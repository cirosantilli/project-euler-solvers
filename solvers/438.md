# Project Euler 438 — Faster Integer-Only Sturm Check (PRS)

This repository contains a pure-Python solution for Project Euler 438.

The main speed-critical part is validating that a candidate polynomial has **exactly one real root**
in each interval \([i, i+1)\) for \(i=1..n\). That’s done with **Sturm’s theorem**.

This update replaces a Fraction-based Sturm chain with a faster **integer-only pseudo-remainder sequence (PRS)**.

---

## Why PRS is faster than Fractions

A direct Sturm chain over \(\mathbb{Q}[x]\) typically needs polynomial divisions with rational coefficients,
which in Python means lots of `Fraction` objects and repeated gcd reductions.

Instead, we compute remainders using *pseudo-remainders*:

- Over \(\mathbb{Q}\), if \(A = QB + R\), then \(R\) generally has rational coefficients.
- Multiply by a big enough integer factor to clear denominators and keep everything integral.
- The pseudo-remainder is an integer polynomial that is a (nonzero) scalar multiple of the true remainder.

Sturm’s theorem is invariant under multiplying any element of the Sturm chain by a **positive** scalar,
so a PRS remainder is perfect for root counting.

---

## The “positive” pseudo-remainder detail

Classical pseudo-remainder uses \( \mathrm{lc}(B)^m \), where \(m = \deg(A)-\deg(B)+1\).
If \(\mathrm{lc}(B)\) is negative and \(m\) odd, that scalar is negative, which would flip signs and break
Sturm sign-variation counts.

So this implementation uses:

\[
|\mathrm{lc}(B)|^m \cdot R
\]

which is always a **positive** multiple of the true remainder, keeping Sturm counts correct.

---

## What stays the same

All other parts of the solver remain as before:

- Enumerate bounded integer values \(p(1),...,p(n)\)
- Use forward differences \(b_m = \Delta^m p(1)\) and divisibility \(b_m \equiv 0 \pmod{m!}\)
- Use the monic constraint \(\Delta^n p(1)=n!\) to derive \(p(n+1)\)
- Build the polynomial exactly in the standard power basis
- Validate with the PRS-based Sturm chain
