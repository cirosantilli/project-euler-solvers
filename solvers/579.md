# Project Euler 579 – Lattice Points in Lattice Cubes

This solution computes:

- **C(n)**: the number of distinct lattice cubes fully contained in the box \([0,n]^3\),
- **S(n)**: the sum over all such cubes of the number of lattice points inside each cube,
- and outputs **S(5000) mod 10^9**.

No external libraries are used.

---

## Core techniques

### 1) Quaternion (Euler–Rodrigues) parametrisation of lattice cube orientations

A lattice cube orientation can be described by three orthogonal integer vectors of equal length,
the cube edge vectors \(u,v,w\).

A classical result is that all such orthogonal frames in \(\mathbb{Z}^3\) can be generated from an
integer quaternion:

\[
q = a + bi + cj + dk,\quad m = a^2+b^2+c^2+d^2
\]

Euler–Rodrigues produces a **3×3 integer matrix** whose columns are orthogonal and satisfy:

- \(u\cdot v = u\cdot w = v\cdot w = 0\)
- \(\|u\|^2=\|v\|^2=\|w\|^2 = m^2\)

So the cube’s side length is the integer \(m\).

---

### 2) Counting placements by coordinate spans

For a fixed orientation \((u,v,w)\), the cube’s coordinates fit into \([0,n]^3\) iff in each axis:

\[
\text{span}_x = |u_x|+|v_x|+|w_x| \le n
\]

Then the number of translations is:

\[
(n-\text{span}_x+1)(n-\text{span}_y+1)(n-\text{span}_z+1)
\]

---

### 3) Lattice points inside a lattice cube

For a lattice cube of side length \(m\), let \(g_i\) be the gcd of the coordinates of the i-th
edge vector (column gcd).

Then the total number of lattice points (boundary + interior) is:

\[
P = m^3 + (m+1)(g_1+g_2+g_3) + 1
\]

This is a 3D analogue of Pick/Ehrhart-type counting.

---

### 4) Avoid enumerating scaled duplicates: polynomial summation over scale factors

A primitive orientation can be scaled by an integer \(t\):

- edges become \(t u, t v, t w\),
- spans scale to \(t\cdot\text{span}\),
- side length becomes \(t m\),
- column gcd sum becomes \(t(g_1+g_2+g_3)\).

For a fixed primitive orientation, valid \(t\) satisfy:

\[
t\cdot\max(\text{span}_x,\text{span}_y,\text{span}_z) \le n
\]

Both placement count and lattice-point count become **polynomials in \(t\)**:

- placements: degree 3
- points: degree 3
- product: degree 6

So summing over \(t\) can be done with **closed-form power sums**
\(\sum t^k\) for \(k\le 6\), avoiding iterating every scale explicitly.

---

### 5) Deduplication via canonical orientation key

Quaternion parametrisations have symmetry (edge permutations and sign flips).
The implementation defensively deduplicates orientations by:

- taking absolute values of all matrix entries,
- sorting columns lexicographically,
- hashing the resulting canonical 9-tuple.
