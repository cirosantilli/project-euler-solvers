# Project Euler 509 ÔÇö Divisor Nim (solution outline)

## Key observation: reduce to a 1ÔÇĹpile impartial game
With three piles, a move affects exactly one pile and the player who cannot move loses.  
ThatÔÇÖs a **disjunctive sum** of three identical impartial games, so **SpragueÔÇôGrundy** applies:

- Each pile size `n` has a Grundy number `g(n)`.
- A triple `(a,b,c)` is losing iff `g(a) XOR g(b) XOR g(c) = 0`.
- Therefore `S(n) = n┬│ Ôłĺ L(n)`, where `L(n)` is the number of losing triples.

So the whole task is:
1) understand `g(n)` for the 1ÔÇĹpile game, then  
2) count how often each Grundy value occurs in `1..n`.

## Grundy number for one pile: `g(n) = v2(n)`
Let `v2(n)` be the exponent of 2 in `n` (the number of times 2 divides `n`).

### 1) Odd `n` have Grundy 0
If `n` is odd, **every proper divisor `d` is odd**, so `n Ôłĺ d` is even.  
That means every reachable position has `v2(nÔłĺd) Ôëą 1`, so **0 is not reachable** and the mex is 0:
`g(n)=0`.

### 2) Even `n = 2^k * m` (with `m` odd) have Grundy `k`
We show:
- Every value `< k` is reachable.
- Value `k` is not reachable.

**Reachability of all `< k`:**  
For any `t < k`, choose the proper divisor `d = n / 2^(kÔłĺt) = 2^t * m`.  
Then:

`n Ôłĺ d = 2^k m Ôłĺ 2^t m = 2^t m (2^(kÔłĺt) Ôłĺ 1)`

and `(2^(kÔłĺt) Ôłĺ 1)` is odd, so `v2(nÔłĺd)=t`. Hence every `t<k` is reachable.

**Impossibility of reaching `k`:**
- If a divisor `d` is **not** divisible by `2^k`, then subtracting it changes the lowest `k` bits, so `v2(nÔłĺd) < k`.
- If `d` **is** divisible by `2^k`, write `d = 2^k d'` with `d'|m` (and `d'` odd).  
  Then `nÔłĺd = 2^k(mÔłĺd')`, and `mÔłĺd'` is even (oddÔłĺodd), so `v2(nÔłĺd) Ôëą k+1`, not `k`.

So mex of `{0,1,...,kÔłĺ1}` is `k`, giving `g(n)=k=v2(n)`.

## Counting losing triples using a frequency table
Let `f[k] = #{1 ÔëĄ x ÔëĄ n : v2(x) = k}`.

This has a simple closed form:

`f[k] = floor(n / 2^k) Ôłĺ floor(n / 2^(k+1))`

To count losing triples, we need:

`v2(a) XOR v2(b) XOR v2(c) = 0`

Equivalently, for every pair `(i,j)` the third value is forced to be `k = i XOR j`:

`L(n) = ╬ú_i ╬ú_j f[i] * f[j] * f[i XOR j]`

Since `v2(x)` is at most `ÔîŐlog2 nÔîő` (Ôëł 47 for the input), we only need an array of size 64 and the double sum is tiny.

Finally:

`S(n) = n┬│ Ôłĺ L(n)` and the problem asks for `S(n) mod 1234567890`.

- Builds `f[k]` via bit shifts (fast exact floors).
- Computes `L(n)` with the `╬ú_i ╬ú_j` formula (optionally modulo `M`).
- Returns `(n┬│ Ôłĺ L(n)) mod M`.
