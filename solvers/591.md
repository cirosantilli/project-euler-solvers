# Project Euler 591 — Best Approximations by Quadratic Integers

This solver computes:

\[
\sum_{2 \le d < 100,\ d\ \text{non-square}} \left|I_d(BQA_d(\pi,10^{13}))\right|
\]

where `BQA_d(x,n)` is the quadratic integer `a + b√d` closest to `x` with
`|a|,|b| ≤ n`.

---

## Key techniques used

### 1) Reduce the 2D search to a 1D modular nearest-point problem

For fixed `b`, the best admissible `a` is the integer closest to:

\[
a \approx \pi - b\sqrt d
\]

Thus the distance depends only on:

\[
\left|\{\pi\} - \{b\sqrt d\}\right|
\]

Let:

- `sqrt(d) = floor(sqrt(d)) + α`, with `α ∈ (0,1)`
- `β = frac(π)`

Then:

\[
\{b\sqrt d\} = \{bα\}
\]

So we must find `b` such that `{bα}` is nearest to `β`.

---

### 2) Continued fractions give good “giant-step” denominators

We use convergents of the continued fraction of `α`:

\[
α \approx p/q
\]

Then `qα ≈ p`, meaning `qα mod 1` is a *small step* `δ`.
This allows scanning candidates of the form:

\[
b = s q + t
\]

and searching in modular arithmetic along arithmetic progressions.

---

### 3) Fixed-point modular arithmetic

To avoid floating precision issues:

- represent fractional values on a large modulus `MOD = 2^k`
- compute distances by integer modular subtraction

This makes comparisons exact and fast.

---

### 4) Careful boundary enforcement

The constraints `|a| ≤ n` and `|b| ≤ n` are enforced by:

- limiting allowed `b` magnitudes based on rounding bounds (`±0.5`)
- checking `a` candidates near the rounded optimum and boundary `±n`

---

## Complexity

The search is approximately `O(√B)` per `d`, with `B ≈ n/√d`.
This is much faster than brute force and works with pure Python,
though still heavy at `n = 10^13`.

---

## Tests

The code includes assertions for all examples given in the Euler statement:

- `BQA_2(π,10) = 6 − 2√2`
- `BQA_5(π,100) = 26√5 − 55`
- `BQA_7(π,10^6) = 560323 − 211781√7`
- `I_2(BQA_2(π,10^13)) = −6188084046055`
