# Project Euler 699 — Triffle Numbers

We need

- \(\sigma(n)\): sum of divisors of \(n\)
- \(T(N)\): sum of all \(n \le N\) such that the reduced fraction \(\sigma(n)/n = a/b\) has denominator \(b = 3^k\) with \(k>0\)

The program computes \(T(10^{14})\) and asserts the two sample values from the statement.

## Key ideas

### 1) Reduce the condition to prime-adic inequalities

Let \(g = \gcd(n,\sigma(n))\). After reducing \(\sigma(n)/n\), the denominator is

\[
b = \frac{n}{g}.
\]

The condition “\(b\) is a power of 3, and \(b>1\)” is equivalent to:

- For every prime \(p \ne 3\): \(v_p(\sigma(n)) \ge v_p(n)\) (all non-3 prime powers cancel completely).
- For \(p=3\): \(v_3(\sigma(n)) < v_3(n)\) (some 3 remains in the denominator, so the exponent is positive).

### 2) Split off the power of 3

Write \(n = 3^a \cdot m\) where \(a = v_3(n) \ge 1\) and \(\gcd(m,3)=1\).

Using multiplicativity of \(\sigma\) over coprime factors:

\[
\sigma(n) = \sigma(3^a)\,\sigma(m).
\]

A crucial simplification is that \(\sigma(3^a)\) is **never divisible by 3**, so:

\[
v_3(\sigma(n)) = v_3(\sigma(m)).
\]

Therefore the “positive power of 3 in the reduced denominator” condition becomes:

- \(v_3(\sigma(m)) < a\)

For primes \(p \ne 3\) that divide \(m\), we need:

- \(v_p(\sigma(3^a)) + v_p(\sigma(m)) \ge v_p(m)\)

So for each fixed \(a\), the search is over valid \(m \le \left\lfloor N/3^a \right\rfloor\).

### 3) Precompute prime-power divisor sums

For each prime \(p\) (excluding 3) and exponent \(e\ge 1\):

- \(p^e\)
- \(\sigma(p^e) = (p^{e+1}-1)/(p-1)\)
- the relevant prime valuations of \(\sigma(p^e)\), stored sparsely

This lets the backtracking update the running valuations of \(\sigma(m)\) quickly.

### 4) Backtracking with monotone pruning

The DFS builds \(m\) by choosing exponents for a fixed finite prime set (valid for the \(10^{14}\) bound).

The state tracks:

- `demand[p] = v_p(m)`
- `supply[p] = v_p(σ(m))` accumulated from chosen prime powers
- `init[p] = v_p(σ(3^a))` fixed for the current \(a\)
- `v3_sigma_m = v3(σ(m))`

Pruning is based on:

- `v3_sigma_m < a` (monotone; once violated, it stays violated)
- For already-fixed primes, `demand[p] <= init[p] + supply[p] + (max possible future supply for p)`

The “max possible future supply” is computed as a suffix-sum bound from the remaining primes’ best contributions, which is fast and effective.

## Correctness checks

The implementation asserts:

- `T(100) = 270`
- `T(10^6) = 26089287`

Then it prints `T(10^14)`.
