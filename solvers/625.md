# Project Euler 625 — Gcd Sum (Python)

This solution computes

\[
G(N)=\sum_{j=1}^{N}\sum_{i=1}^{j}\gcd(i,j)
\]

and outputs \(G(10^{11}) \bmod 998244353\).

## Key ideas

### 1) Turn the gcd sum into a totient sum

For a fixed \(j\), group terms by \(d=\gcd(i,j)\).  
Write \(j=d\cdot k\) and \(i=d\cdot m\). Then \(\gcd(m,k)=1\) and \(1\le m\le k\), so there are \(\varphi(k)\) such \(m\).

That gives:

\[
\sum_{i=1}^{j}\gcd(i,j)=\sum_{d\mid j} d\;\varphi\!\left(\frac{j}{d}\right)
\]

Summing over \(j\le N\) and swapping the order with \(j=d\cdot k\) yields:

\[
G(N)=\sum_{k=1}^{N}\varphi(k)\sum_{d\le N/k} d
     =\sum_{k=1}^{N}\varphi(k)\cdot T\!\left(\left\lfloor\frac Nk\right\rfloor\right)
\]

where \(T(x)=x(x+1)/2\) is the triangular number.

### 2) Hyperbola method (split at \(\sqrt{N}\))

The values \(\left\lfloor N/k\right\rfloor\) change slowly and can be handled with a classic “hyperbola” split:

- handle small \(k\le \lfloor\sqrt{N}\rfloor\) directly using the precomputed \(\varphi(k)\)
- handle the remaining part with the *summatory totient* function

\[
\Phi(n)=\sum_{k\le n}\varphi(k)
\]

so ranges of \(\varphi\) values can be obtained from prefix sums.

The function `G(n)` in `main.py` implements this split in \(O(\sqrt{N})\) arithmetic operations plus the cost of evaluating \(\Phi(\cdot)\) a relatively small number of times.

### 3) Fast summatory totient via recursion + memoization

We need \(\Phi(n)\) for large \(n\) (up to \(10^{11}\)).

A useful identity is:

\[
\sum_{k\le n}\varphi(k)\left\lfloor\frac{n}{k}\right\rfloor = \sum_{m=1}^{n} m = T(n)
\]

Using the fact that \(\left\lfloor n/k\right\rfloor\) takes only \(O(\sqrt{n})\) distinct values, we can write a recursion for \(\Phi(n)\) that:
- uses a sieve to precompute exact \(\Phi(n)\) for all \(n \le L\)
- uses memoization for all larger queries
- only recurses when \(n/m > L\), which keeps the number of “large” states small

With \(L=10^7\), this is comfortably fast for \(N=10^{11}\) in pure Python.

### 4) Implementation details that matter in Python

- **Linear sieve** for \(\varphi\) to keep the sieve \(O(L)\).
- Use `array('I')` and `bytearray` to keep memory usage reasonable.
- Minimize `% MOD` operations inside tight loops (accumulate, then reduce).
