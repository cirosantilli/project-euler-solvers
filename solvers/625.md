# Project Euler 625 — Gcd Sum (faster algorithm)

We need:

\[
G(N)=\sum_{j=1}^{N}\sum_{i=1}^{j}\gcd(i,j)
\]

and output \(G(10^{11}) \bmod 998244353\).

---

## Main technique 1: Reduce to a totient-weighted sum

For fixed \(j\), group \(i\) by \(d=\gcd(i,j)\). Write \(j=d\cdot k\) and \(i=d\cdot m\).  
Then \(\gcd(m,k)=1\), and there are \(\varphi(k)\) such \(m\).

This yields:

\[
G(N)=\sum_{k=1}^{N}\varphi(k)\,T\!\left(\left\lfloor\frac{N}{k}\right\rfloor\right),
\quad
T(x)=\frac{x(x+1)}2
\]

---

## Main technique 2: Hyperbola / quotient grouping

The quotient \(q=\lfloor N/k\rfloor\) is constant on intervals \([l,r]\) where:

- \(q = N//l\)
- \(r = N//q\)

So we can compute the outer sum in about \(2\sqrt{N}\) blocks:

\[
\sum_{k=l}^{r}\varphi(k)=\Phi(r)-\Phi(l-1)
\]

where \(\Phi(n)=\sum_{k\le n}\varphi(k)\) is the **summatory totient**.

---

## Main technique 3: Dujiao summatory totient with memoization

A standard identity leads to the recursion (evaluated with quotient grouping):

\[
\Phi(n)=T(n)-\sum_{l=2}^{n}(r-l+1)\,\Phi\!\left(\left\lfloor\frac{n}{l}\right\rfloor\right),
\quad r=\left\lfloor\frac{n}{\lfloor n/l\rfloor}\right\rfloor
\]

We memoize \(\Phi(n)\) for large values and use a sieve for \(n\le L\).

---

## Critical performance choices

1) **Sieve cutoff** \(L\approx N^{2/3}\) (classic Dujiao sweet spot).  

2) **Warm memo once**: compute \(\Phi(N)\) *before* the outer loop.  
   This populates the memo with every large \(\Phi\) value the outer sum will ask for, preventing many expensive independent Phi computations.

3) **Tight modular arithmetic**: keep values in \([0,\text{MOD})\) with branch-only adjustments in the hot loop.
