# Project Euler 625 — Gcd Sum (fast pure Python)

We need to compute

\[
G(N)=\sum_{j=1}^{N}\sum_{i=1}^{j}\gcd(i,j)
\]

and output \(G(10^{11}) \bmod 998244353\).

This solution is tuned for performance in pure Python (no third‑party libraries).

---

## 1) Reduce the double sum using Euler’s totient

For a fixed \(j\), group terms by \(d=\gcd(i,j)\).
Write \(j=d\cdot k\) and \(i=d\cdot m\). Then \(\gcd(m,k)=1\) and \(1\le m\le k\), so there are \(\varphi(k)\) valid \(m\).

\[
\sum_{i=1}^{j}\gcd(i,j)=\sum_{d\mid j} d\,\varphi\!\left(\frac{j}{d}\right)
\]

Summing over \(j\le N\) and substituting \(j=d\cdot k\) gives:

\[
G(N)=\sum_{k=1}^{N}\varphi(k)\sum_{d\le N/k} d
     =\sum_{k=1}^{N}\varphi(k)\,T\!\left(\left\lfloor\frac Nk\right\rfloor\right)
\]

where \(T(x)=x(x+1)/2\) is the triangular number.

---

## 2) Hyperbola grouping makes the outer sum ~\(2\sqrt{N}\)

The quotient \(q=\left\lfloor N/k\right\rfloor\) only takes about \(2\sqrt{N}\) distinct values.
For each segment \([l,r]\) where \(N//k\) is constant:

- \(q = N//l\)
- \(r = N//q\)
- all \(k\in[l,r]\) share this \(q\)

So:

\[
\sum_{k=l}^{r}\varphi(k)=\Phi(r)-\Phi(l-1)
\]

with \(\Phi(n)=\sum_{k\le n}\varphi(k)\).

Thus:

\[
G(N)=\sum_{\text{segments }[l,r]} \big(\Phi(r)-\Phi(l-1)\big)\,T(q)
\]

This is implemented with a `while l <= N:` loop and integer-division grouping.

---

## 3) Fast summatory totient \(\Phi(n)\) via Dujiao recursion + memoization

A classic identity is:

\[
\sum_{d\mid m}\varphi(d)=m
\]

Summing for \(m\le n\):

\[
T(n)=\sum_{m\le n} m = \sum_{d\le n}\varphi(d)\left\lfloor\frac{n}{d}\right\rfloor
\]

Rearranging and grouping equal quotients gives the Dujiao-style recurrence:

\[
\Phi(n)=T(n)-\sum_{l=2}^{n} (r-l+1)\,\Phi\!\left(\left\lfloor\frac{n}{l}\right\rfloor\right)
\]

where \(r = \left\lfloor n / \left\lfloor n/l\right\rfloor \right\rfloor\).

Because \(\left\lfloor n/l\right\rfloor\) changes only \(O(\sqrt{n})\) times, that sum is computed in grouped blocks.
Memoization ensures each distinct \(\Phi(x)\) is computed once.

We also sieve \(\varphi(k)\) up to a moderate limit (5,000,000) and store prefix sums there, so all \(\Phi(n)\) queries for \(n\le\) limit are O(1).
