# Project Euler 660 — Solution Summary

This solution searches for **n-pandigital 120° integer triangles** \((a,b,c)\) for bases \(9 \le n \le 18\), where:

- The triangle has a 120° angle opposite the largest side \(c\), so  
  \[
  c^2 = a^2 + ab + b^2
  \]
- Writing \(a\), \(b\), and \(c\) in **base \(n\)** uses **every digit** \(0,1,\dots,n-1\) **exactly once** across the three representations (no leading zeros).

The final program prints the required sum of all such largest sides \(c\) for \(n=9,\dots,18\).

---

## Main Techniques

### 1) Digit-by-digit construction (Hensel-style lifting)

The key idea is to build \(a\), \(b\), and \(c\) **from least significant digit to most significant digit** in base \(n\).

Define:
\[
F(a,b,c)=a^2+ab+b^2-c^2
\]

After fixing the first \(p\) base-\(n\) digits of each number, we maintain that:
\[
F(a,b,c) \equiv 0 \pmod{n^p}
\]

When we extend by one more digit (position \(p\)), the constraint for \(p\ge 1\) reduces to a **linear congruence modulo \(n\)**, which is much cheaper than re-checking the full quadratic equation at every step.

---

### 2) Carry tracking

Instead of recomputing squares constantly, we track a “carry”:
\[
G_p = \frac{F}{n^p}
\]
and update it incrementally when we append a digit. When all digits are placed, the triangle is valid exactly when the final carry becomes **0**.

---

### 3) Bitmask representation of remaining digits

Digits \(0..n-1\) must be used exactly once, so the remaining available digits are stored as a single integer bitmask.

- Checking if a digit is available: bit test
- Removing a digit: XOR or bit clear
- Iterating available digits: repeated `lsb = mask & -mask`

This is significantly faster than sets/lists for \(n \le 18\).

---

### 4) Solving the congruence with modular inverses

At each digit position (for \(p \ge 1\)) we solve a linear congruence like:
\[
\alpha d_a + \beta d_b + \delta d_c \equiv t \pmod n
\]

Whenever a coefficient is coprime to \(n\), we can use a **modular inverse** to determine that digit **uniquely** (reducing branching).

If a coefficient is not invertible mod \(n\), we use a precomputed “residue → digit mask” table to enumerate the few possibilities efficiently.

---

### 5) Symmetry reduction

For partitions where \(a\) and \(b\) have the same number of base-\(n\) digits, swapping them produces the same triangle. We break this symmetry early by enforcing an ordering on the least significant digits.
