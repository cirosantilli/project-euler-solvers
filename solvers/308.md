# Project Euler 308 — Solution Notes

This problem runs John Conway’s famous **FRACTRAN** program `PRIMEGAME` starting from the seed `2` and asks:

> How many FRACTRAN iterations occur before the machine first produces the value `2^(10001st prime)`?

A direct simulation is hopeless because the intermediate integers become enormous and the number of iterations is ~`10^15`.

## Key ideas used in `main.py`

### 1) View FRACTRAN as a register machine (prime-exponent encoding)
A FRACTRAN state is a single integer, but its **prime factorization** encodes many registers:

- The exponent of `2` is one register,
- the exponent of `3` is another,
- etc.

Each fraction `a/b` acts like:

- *test* that every prime in `b` has enough exponent,
- then subtract those exponents (divide by `b`) and add the exponents from `a` (multiply by `a`).

This avoids dealing with huge integers conceptually, and exposes the program’s control structure.

### 2) Identify “boundary states” that matter
For Conway’s `PRIMEGAME` with seed `2`, the computation repeatedly reaches states that contain **only primes 2 and 7**:

\[
2^N \cdot 7^{m}
\]

At these boundaries, the program has finished “checking” a particular integer `N`.
A classic analysis of `PRIMEGAME` shows that the boundary after finishing `N` is:

\[
2^N \cdot 7^{b(N)-1}
\]

where `b(N)` is the **largest proper divisor** of `N` (and `b(N)=1` if `N` is prime). Therefore:

- `N` is prime **iff** the boundary is a pure power of two (`7`-exponent is zero).
- The primes appear as exponents of powers of two in increasing order.

So instead of simulating every FRACTRAN step, we can sum the exact cost of advancing from boundary `N-1` to boundary `N`.

### 3) Closed-form cost per checked number `N`
Let `b(N)` be the largest proper divisor of `N`.
For each boundary transition `(N-1) -> N`, the number of FRACTRAN iterations is:

\[
\text{cost}(N) = (N-1) + (6N+2)(N-b(N)) + 2\sum_{d=b(N)}^{N-1} \left\lfloor\frac{N}{d}\right\rfloor + (b(N-1)-1).
\]

- The big term with the floor-sum comes from the quotient/remainder loops inside Conway’s algorithm.
- The final `+(b(N-1)-1)` term is specific to the **original Euler 308 fraction list** (it includes `1/7`), and accounts for a countdown of the previous `7`-exponent.

Finally:

\[
\text{steps to reach }2^P = \sum_{N=2}^{P} \text{cost}(N).
\]

### 4) Fast computation of `b(N)` for all `N`
We need `b(N)` up to `P = 10001st prime`.
Compute an array of **smallest prime factors** using a sieve, then:

- if `N` is prime: `b(N)=1`,
- otherwise: `b(N)=N / spf(N)`.

This is `O(P log log P)` time and `O(P)` memory.

### 5) Fast computation of the floor-division sum
The sum

\[
\sum_{d=lo}^{hi} \left\lfloor\frac{N}{d}\right\rfloor
\]

is computed in `O(\sqrt{N})` time by grouping ranges of `d` where `N//d` is constant.
This makes the overall runtime roughly `O(P \sqrt{P})` in the worst case, but very fast for `P ≈ 10^5`.

## Result
Running `main.py` with no input prints the Project Euler #308 answer:

- **1539669807660924**

(If you provide an integer `n` on stdin, the program computes the iteration count for `2^(nth prime)`.)
