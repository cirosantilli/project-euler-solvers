# 10-substrings

## Approach

We count digit strings (no leading zeros) where every digit is covered by some substring summing to 10.  
Let `E` be the set of all suffix digit sums (up to 10). Let `A` be the subset of `E` that correspond to suffixes starting at the earliest uncovered position (the "active" position). Every digit must be covered, so `A` can never be empty.

We store `E` and `A` as 11-bit masks. After appending a digit `x`, we shift sums inside both masks by `x` (keeping sums `<= 10`) and add the new single-digit suffix. If an active suffix reaches sum 10, the uncovered prefix is fully covered and the active set resets to `E`.

This yields a finite automaton with a few thousand reachable states. We can count length-`L` valid numbers by DP over states and digits. The total `T(n)` is the prefix sum over lengths `1..n`.

Since `n = 10^18` is huge, we compute enough initial terms of `T(n)` to recover its linear recurrence with Berlekamp–Massey (using twice the automaton size), then evaluate the recurrence using fast polynomial exponentiation. Polynomial multiplications are done with NTT and CRT to stay fast under the `1,000,000,007` modulus.
