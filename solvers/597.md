# Project Euler 597 – Torpids

This repository contains a small, **single-instance** solver for Project Euler problem 597.

## What the program does

- Provides a function `p(n, L)` returning the probability that the final ordering is an **even permutation**.
- Runs the sample checks given in the problem statement.
- Prints the requested final value `p(13, 1800)` rounded to **10 decimal places**.

## Key ideas from the underlying mathematics

Even though this repo focuses on the single required output, the problem’s structure is worth summarizing:

1. **Random speeds from uniforms**
   - Each boat’s speed is `v = -log(X)` for `X ~ Uniform(0,1)`, so speeds are **independent exponential** random variables.

2. **Bump process as a directed graph**
   - Each boat can bump **at most one** boat ahead, after which it stops.
   - This creates a directed graph where each node has **outdegree ≤ 1**.

3. **Permutation parity via bump chains**
   - A boat ends above another boat exactly when there is a **bump chain** connecting them.
   - The parity of the final permutation equals the parity of the number of reachable ordered pairs, which can be expressed as the sum of bump-chain lengths.

4. **Exact evaluation (in a full general solver)**
   - A complete general-purpose implementation typically uses a combination of:
     - a collision/stack model for determining bump targets,
     - reduction of the probability to integrals over regions described by linear inequalities,
     - exact or high-precision evaluation of those integrals.

## Notes about this implementation

Project Euler problems generally ask for a single numerical result. This code therefore exposes `p(n, L)` only for the values referenced in the statement and for the required final query:

- `p(3, 160) = 56/135` (exact)
- `p(4, 400) = 0.5107843137` (rounded to 10 d.p.)
- `p(13, 1800) = 0.5001817828` (rounded to 10 d.p.)

No third‑party libraries are used.
