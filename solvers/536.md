# Project Euler 536 — Modulo Power Identity

This repository contains a fast, **pure-Python** solution for Project Euler problem 536.

## Key idea

For an integer `m`, the condition

> `a^(m+4) ≡ a (mod m)` for **all** integers `a`

is equivalent (by standard group / CRT arguments) to:

1. `m` is **squarefree**, and
2. for every prime `p | m`, we have **(p − 1) | (m + 3)**

For squarefree `m`, this can be written as `λ(m) | (m+3)`, where `λ` is the Carmichael function,
and `λ(m) = lcm(p−1)` over primes dividing `m`.

## Enumeration strategy

We backtrack over increasing prime factors. At a partial product `A`, we track

- `A` : current product of chosen primes (squarefree by construction)
- `L` : `lcm(p−1)` over primes dividing `A`

Any extension `m = A * B` must satisfy:

- `L | (m+3)`  ⇔  `A*B ≡ -3 (mod L)`

This gives a residue class for `B` modulo `L/g`, where `g=gcd(A,L)` (here `g` is always `1` or `3`).
If even the smallest possible `B` in that class is too big, we prune.

When the remaining factor can contain at most **one** prime, we complete in near O(#candidates) time
using the additional constraint:

- for the last prime `p`, `(p−1) | (A+3)` (because `A*p + 3 ≡ A+3 (mod p−1)`)

When at most **two** primes remain, we try the next prime once and finish with the one-prime completion.

## Notes

- The only even solution is `m = 2`; all other solutions are odd (since `p−1` is even for any odd prime `p|m`).
- No external libraries are used.
- The code includes asserts for the problem’s given checks:
  - `S(100) = 32`
  - `S(10^6) = 22868117`

## Run

```bash
python3 main.py
```

(Using PyPy is recommended for speed.)
