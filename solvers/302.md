# Project Euler 302 — Strong Achilles Numbers (Python)

This solution counts **strong Achilles numbers** below \(10^{18}\).

A number \(n\) is:

- **Powerful** if for every prime \(p \mid n\), we have \(p^2 \mid n\).
- A **perfect power** if \(n = a^k\) for integers \(a>1, k>1\).
- **Achilles** if it is powerful but not a perfect power.
- **Strong Achilles** if \(n\) is Achilles **and** \(\varphi(n)\) is also Achilles.

The expected final answer is:

> **1170060**

---

## Core Techniques Used

### 1) Exponent–GCD Characterization of Perfect Powers
If

\[
n=\prod p_i^{e_i},
\]

then \(n\) is a **perfect power** iff \(\gcd(e_1,e_2,\dots)>1\).

So Achilles means:

- all \(e_i \ge 2\) (powerful)
- \(\gcd(e_i)=1\) (not a perfect power)

---

### 2) Incremental Totient Factorization
Using:

\[
\varphi(n)=\prod p_i^{e_i-1}(p_i-1),
\]

we can track the prime-exponent multiset of \(\varphi(n)\) while building \(n\).

When we add a prime-power factor \(p^e\):

- Add exponent \((e-1)\) to prime \(p\)
- Factorize \((p-1)\) and add those exponents too

This lets us test whether \(\varphi(n)\) is:

- **powerful** (all exponents \(\ge2\))
- **not a perfect power** (\(\gcd(\text{exponents})=1\))

---

### 3) Depth-First Search Over Prime Factorizations
We enumerate numbers:

\[
n = \prod p_i^{e_i}
\]

by choosing primes in increasing order and exponents \(e_i \ge 2\),
ensuring \(n < 10^{18}\).

We maintain:

- `current_n` = current product
- `gcd_n` = gcd of exponents so far
- `phi_exps` = dict of prime → exponent in \(\varphi(n)\)
- `pending` = primes currently having exponent **1** in `phi_exps`

---

### 4) “Pending primes” Pruning
A prime exponent of **1** in \(\varphi(n)\) is illegal (totient must be powerful),
so such primes must be fixed later by future factors.

If a pending prime exceeds the maximum possible future prime, we prune early.

---

### 5) Safe Lower-Bound Pruning via LCM Incompatibility
Two pending primes \(a,b\) can only be fixed simultaneously by a future prime \(r\) if:

\[
r \equiv 1 \pmod{\mathrm{lcm}(a,b)}
\]

If \(\mathrm{lcm}(a,b)\) exceeds the maximum possible future prime,
then **they must require distinct future primes**.

We greedily build a subset of pending primes that are pairwise incompatible and derive a
safe multiplicative lower bound for remaining budget:

\[
\prod (q+1)^2
\]

If this lower bound exceeds remaining capacity, we prune.
