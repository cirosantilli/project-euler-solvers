# Project Euler 353

## Approach

- Integer stations on the sphere satisfy `x^2 + y^2 + z^2 = r^2`.
- For each fixed `z`, the remaining equation is `x^2 + y^2 = (r - z)(r + z)`. We factor this value and generate all integer representations as a sum of two squares using Gaussian integer multiplication. This avoids brute-forcing the whole cube.
- The travel risk between two stations depends only on their dot product, so we use Dijkstra's algorithm on the station graph with edge weight `(acos(dot / r^2) / pi)^2`.
- To keep the graph sparse, we only connect each station to points in nearby `z`-layers (a window proportional to `sqrt(r)` in both directions). For each `z`-layer we keep only a small set of nearest angular neighbors (and all points on the same layer).
- The target sum is computed for `r = 2^n - 1` for `n = 1..15`, and the result is printed with 10 digits after the decimal point.
