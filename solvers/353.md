# Project Euler 353 — Risky Moon (Implementation Notes)

This implementation focuses on speed in pure Python (no third‑party libraries), by avoiding the complete graph and by pruning Dijkstra aggressively.

## Key techniques

### 1) Generate lattice points without scanning the whole cube
We need all integer solutions to:

\[
x^2 + y^2 + z^2 = r^2
\]

Instead of scanning all \((x,y)\) and solving for \(z\), we generate points via **Hurwitz quaternion parameterization**:

- Integer quaternions: \(a^2+b^2+c^2+d^2=r\)
- Half‑integer (Hurwitz) quaternions: odd \(a,b,c,d\) with \(a^2+b^2+c^2+d^2=4r\)

From a quaternion \(q\), the vectors \(q\,i\,\bar q\), \(q\,j\,\bar q\), \(q\,k\,\bar q\) give integer points on the sphere.

We collect **primitive** points (those with \(\gcd(x,y,z)=1\)) and then obtain **all** lattice points on radius \(r\) using divisor scaling:
for each \(d\mid r\), scale primitive points on radius \(d\) by \(r/d\).

### 2) Edge weights from dot products (with caching)
For points \(p,q\) on a sphere of radius \(r\):

\[
\cos(\theta)=\frac{p\cdot q}{r^2},\quad
\text{risk}=\left(\frac{\theta}{\pi}\right)^2
\]

Computing `acos()` is expensive, so risks are cached by the integer dot product \(p\cdot q\).

### 3) Sparse neighbor graph using 3D bucket hashing
The full graph is complete, but quadratic risk strongly favors splitting long arcs.
So we only consider edges with chord length \(\le D\) (a tunable threshold).

To find candidates quickly, points are put into 3D buckets of size `D`.
From a node, only the 27 adjacent buckets are scanned.

### 4) Bounded Dijkstra (branch‑and‑bound)
During Dijkstra:

- We maintain `best`, the cheapest **known** path to the goal so far.
- Any node popped with distance \(\ge best\) can be ignored (and we can terminate early).
- Any relaxation producing `nd >= best` is skipped.

To tighten `best` early, we first run a small best‑first search (A*‑like, non‑admissible heuristic) that quickly finds *some* feasible path. This does **not** affect correctness because it only supplies an upper bound used for pruning.

### 5) Adaptive choice of `D`
We start with a local `D ≈ 2√r` and increase until the sparse graph is connected.
Then we refine lightly (small factor) to catch improvements without making neighbor scans explode.
