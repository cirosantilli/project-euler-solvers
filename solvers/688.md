# Project Euler 688 – Approach Summary

## Key observations

### 1) Closed form for `f(n, k)`
If the smallest pile has size `m`, then the smallest possible distinct configuration is:

`m, m+1, ..., m+k-1`

Its sum is:

`k*m + k(k-1)/2`

So `m` is feasible iff:

`k*m + k(k-1)/2 ≤ n`

The maximum such `m` is therefore:

`f(n,k) = max(0, floor((n - k(k-1)/2) / k))`

(And `f(n,k)=0` exactly when `n < k(k+1)/2`.)

### 2) Turning `S(n)` into a weighted lattice-point sum
Using the identity `x = Σ_{m=1..x} 1`, we can rewrite `f(n,k)` as a count of how many `m ≥ 1` satisfy the inequality above.
Summing over `n` from `1..N` turns the problem into summing an arithmetic progression for each feasible `(k,m)`:

`S(N) = Σ_{k≥1} Σ_{m≥1, k*m + k(k-1)/2 ≤ N} (N + 1 - (k*m + k(k-1)/2))`

For a fixed `k`, define:

`q = floor((N - k(k-1)/2) / k)`

Then `m` ranges from `1..q`, and the inner sum has a closed form:

`contribution(k) = q*(N + 1 - k(k-1)/2) - k*q*(q+1)/2`

So:

`S(N) = Σ_{k=1..kmax} contribution(k)`

where `kmax` is the largest `k` such that `k(k+1)/2 ≤ N`.

### 3) Modular arithmetic
The required output is `S(10^16) mod 1_000_000_007`.
Because division by `2` appears, we use the modular inverse of `2` (which is `(MOD+1)//2` for an odd prime modulus).

## Implementation notes

- `kmax` is computed via the quadratic formula with `isqrt`.
- The triangular term `k(k-1)/2` is maintained incrementally to avoid repeated multiplications.
- The code includes asserts for all sample values given in the problem statement.
