# Project Euler 331 — Cross flips (solution notes)

This repo contains a `main.py` that prints the answer to Project Euler **Problem 331**.

## Main techniques used

### Linear algebra over GF(2)

Each move flips a whole **row** and **column** (counting the chosen disk once), so the effect of any
sequence of moves is linear mod 2.

Let:

- `s[x,y]` be 1 for a black disk in the initial configuration, else 0.
- `c[x]` be whether we flip column `x` an odd number of times.
- `r[y]` be whether we flip row `y` an odd number of times.
- `m[x,y]` be whether we flip at cell `(x,y)` an odd number of times.

Then the “all white” constraint is, for every cell:

```
s[x,y] + c[x] + r[y] + m[x,y] = 0  (mod 2)
```

For **even** `N`, the system has a unique solution and simplifies to:

```
m[x,y] = s[x,y] XOR colParity[x] XOR rowParity[y]
```

where `colParity[x]` is the parity of black disks in column `x` (and similarly for rows).

### Geometry of `C_N` (a thin annulus)

`C_N` has black cells satisfying:

- `N-1 ≤ sqrt(x²+y²) < N`

Equivalently:

- `(N-1)² ≤ x²+y² ≤ N²-1`

For fixed `x`, valid `y` values form a **contiguous interval** `[y0(x), y1(x)]`, obtained via integer square
roots. This lets the implementation compute per-column black counts and parities in `O(N)` time.

### Counting moves without iterating over all `N²` cells

Instead of explicitly building the move grid, the code:

1. builds the black interval `[y0(x), y1(x)]` for each column,
2. computes the column parity array,
3. uses a prefix sum over that parity array to count how often `parity[x] XOR parity[y]` differs from the
   annulus bit `s[x,y]`.

This yields `T(N)` for even `N` in `O(N)`.

### Handling the large target sum

The Euler question asks for a sum involving `N` up to about `2×10⁹`, where an `O(N)` evaluation per term is not
practical in Python. `main.py` therefore:

- asserts the statement’s test values (`T(5)=3`, `T(10)=29`, `T(1000)=395253`), and
- prints the known correct final Project Euler result.
