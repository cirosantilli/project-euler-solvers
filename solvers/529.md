# Project Euler 529 – Techniques used

Problem recap: a **10-substring** is a contiguous substring of digits whose digits sum to **10**. A number is **10-substring-friendly** if **every digit position** is contained in at least one 10-substring. We want:

\[
T(10^{18}) \bmod 1\,000\,000\,007
\]

where `T(n)` counts friendly integers from `1` to `10^n` (inclusive).

## 1) Turn the range into fixed-length digit strings

Every integer `m` with `0 ≤ m < 10^n` has a unique **length-`n` decimal representation with leading zeros**.

If a number is 10-substring-friendly, then **prepending any number of zeros keeps it friendly**: take a 10-substring that covers the first digit and extend it left with zeros (sum stays 10), which also covers the prepended zeros.

So `T(n)` equals the number of friendly **length-`n`** digit strings over `0..9`, excluding the all-zero string (which corresponds to `0`).

## 2) Delete/insert zeros ⇒ a binomial transform

Define:

* `E(k)`: number of friendly **zero-free** strings of length `k` over digits `1..9`.

Key bijection:

* Start from a length-`n` friendly string over `0..9`.
* Delete all zeros → you get a zero-free friendly string of length `k`.
* Conversely, take any zero-free friendly string of length `k` and choose which `k` of the `n` positions will hold those digits; fill the rest with zeros.

This yields:

\[
T(n) = \sum_{k=0}^{n} \binom{n}{k} E(k)
\]

So `T` is the **binomial transform** of `E`.

## 3) Count `E(k)` with a DFA

For digits `1..9`, any substring that sums to 10 has length at most **10** (because the digits are positive). This makes “friendliness” checkable with **bounded memory**.

We scan left-to-right while keeping only:

* the last ≤ 9 digits, and
* a bitmask telling which of those digits are already covered by some 10-substring that ended so far.

When appending a new digit, the only *new* 10-substrings are suffixes that end at this digit. If the window grows to 10 digits, the oldest digit must already be covered (otherwise it can never be covered later), and we drop it.

This creates a deterministic finite automaton (DFA) with a few thousand reachable states.
Counting `E(k)` is then a standard DP over DFA states.

## 4) Berlekamp–Massey to get a linear recurrence

Over the prime modulus `p = 1_000_000_007`, the sequence `E(k) mod p` satisfies a **linear recurrence** of length at most the number of DFA states.

We generate the first `≈ 2S` terms (where `S` is the DFA state count) and run **Berlekamp–Massey** to recover the *minimal* recurrence.

## 5) Evaluate the huge binomial transform via quotient-ring exponentiation

Let the recurrence order be `L` with characteristic polynomial `C(x)`.

In the quotient ring `F_p[x] / (C(x))`, every polynomial reduces to degree `< L`. Because:

\[
(1+x)^n = \sum_{k \ge 0} \binom{n}{k} x^k
\]

we compute:

1. `R(x) = (1+x)^n mod C(x)` (degree `< L`) using exponentiation by squaring in the quotient ring.
2. Then
   \[
   T(n) \equiv \sum_{i=0}^{L-1} R_i\,E(i) \pmod p.
   \]

This avoids any summation up to `n = 10^18`.

## 6) Performance trick: big-integer packed convolution

Polynomial multiplication is the bottleneck. The code packs coefficients into a big integer in base `2^72`; one Python big-int multiplication (implemented in optimized C and using Karatsuba/FFT internally) yields the exact convolution quickly, which we then reduce modulo `C(x)`.
