# Project Euler 529 – Solution notes

We call a digit string **10-substring-friendly** if **every position** lies inside **some contiguous substring whose digits sum to 10**.

The problem asks for `T(10^18) mod 1,000,000,007`, where `T(n)` counts friendly integers from `1` to `10^n` (inclusive).

## 1) Turn the problem into fixed-length strings

Every integer `m` with `0 ≤ m < 10^n` can be written as an **n-digit string with leading zeros**.

For `n ≥ 1`, the number `10^n = 1` followed by `n` zeros is **not** friendly (there is no way to include the leading `1` in a sum-10 substring), so:

- `T(n)` equals the number of friendly **length-n digit strings** over `0..9` (leading zeros allowed).

This removes the “up to `10^n`” complication.

## 2) Separate out zeros: a binomial transform

Let:

- `E(k)` = number of friendly length-`k` strings over digits `1..9` (no zeros).

A key property (true because digits `1..9` are strictly positive) is:

- If a string over `1..9` is friendly, then **inserting zeros anywhere** keeps it friendly.
  Any sum-10 substring can be extended to cover inserted zeros without changing the sum.

Conversely:

- If a friendly `0..9` string has its zeros deleted, the remaining `1..9` string is still friendly.

Therefore, choosing which `k` positions are nonzero and filling them with a zero-free friendly string gives:

\[
T(n) = \sum_{k=0}^{n} \binom{n}{k} E(k)
\]

So `T` is the **binomial transform** of `E`.

## 3) Count `E(k)` with a DFA (finite state machine)

For digits `1..9`, any substring summing to 10 has length at most 10, so “friendliness” is a **local** condition.
We scan left-to-right and keep a bounded amount of information about:

- the maximal suffix whose digit-sum is `< 10`, and
- which digits in that suffix have already been “covered” by a completed sum-10 substring.

This yields a deterministic finite automaton (DFA) with a few thousand reachable states.
Counting accepted strings of length `k` becomes a standard DP on the DFA.

## 4) Compress the DP into a linear recurrence (Berlekamp–Massey)

Over a prime modulus `p = 1,000,000,007`, the sequence `E(k) mod p` satisfies a **linear recurrence**.
We obtain the recurrence using **Berlekamp–Massey** (a black-box “minimal recurrence” algorithm) from the first few thousand DFA DP values.

In this repository, the resulting recurrence coefficients and initial values are embedded directly in `main.py` (computed once offline).

## 5) Evaluate the huge binomial transform via polynomial arithmetic

Let the recurrence order be `L` with characteristic polynomial `C(x)`.

In the quotient ring `F_p[x] / (C(x))`, every polynomial reduces to degree `< L`.

Because:

\[
(1+x)^n = \sum_{k \ge 0} \binom{n}{k} x^k
\]

we can compute:

- `R(x) = (1+x)^n mod C(x)` (degree `< L`),
- then

\[
T(n) \equiv \sum_{i=0}^{L-1} R_i E(i) \pmod p
\]

So we never sum up to `n` directly; we only work with `L` coefficients.

## 6) Make it fast: big-int packed convolution + fast modular reduction

The bottleneck is multiplying degree-`<L` polynomials many times (for exponentiation).

Two optimizations are used:

1. **Exact convolution via big integers**  
   Coefficients are packed into a big integer in base `2^72`.  
   One Python integer multiplication (implemented in C with Karatsuba/FFT) produces the whole convolution quickly.

2. **Fast reduction mod `C(x)` using a precomputed inverse**  
   Modular reduction is done by polynomial division in reversed form, using a Newton-computed inverse of the reversed modulus.

Together, this makes exponentiation feasible in pure Python without external libraries.

## Sanity checks

The problem statement includes:

- `T(2) = 9`
- `T(5) = 3492`

`main.py` includes asserts for these values before computing the final answer.
