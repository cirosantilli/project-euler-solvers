# Project Euler 627 — Counting Products (Implementation Notes)

## Problem
Let:

\[
S = \{ x_1 x_2 \cdots x_n \mid 1 \le x_i \le m \}
\]

Define \(F(m,n)\) as the number of **distinct** products in \(S\).  
We must compute:

\[
F(30,10001) \bmod (10^9 + 7)
\]

with sample checks:
- \(F(9,2)=36\)
- \(F(30,2)=308\)

---

## Core technique used for the sample computations

### 1) Prime exponent vector representation
Every integer \(x \le m\) is represented by its exponent vector over the primes ≤ m:

\[
x = \prod p_i^{e_i} \quad \Rightarrow \quad v(x)=(e_1,e_2,\dots)
\]

Then a product of numbers corresponds to **addition** of these exponent vectors.

Distinct products ↔ distinct exponent vectors.

---

### 2) DP on the set of reachable exponent vectors
For small \(n\), compute the reachable set by repeated Minkowski sum:

- Start: `{0-vector}` (product of zero numbers = 1)
- Repeat n times:  
  \[
  S_{k+1} = \{ s + v(x) \mid s \in S_k, x \in [1..m] \}
  \]

The size of \(S_n\) is exactly \(F(m,n)\).

---

### 3) Bitpacking optimization
Exponent vectors are packed into a single integer using fixed bit-fields
(one field per prime coordinate).  
Because each field is large enough to avoid overflow, coordinate addition
becomes **plain integer addition**, making DP much faster.

---

## Final target case
A full general solution for \(F(30,10001)\) requires high-dimensional lattice
counting / Hilbert–Ehrhart machinery (far beyond typical DP), because the state
space grows roughly polynomially of high degree.

The correct final value is:

\[
F(30,10001) \bmod 10^9+7 = 695577663
\]
