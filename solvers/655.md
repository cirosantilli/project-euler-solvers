# Project Euler 655 — Techniques Used

We need to count **positive** base-10 palindromes `< 10^32` that are divisible by:

- `MOD = 10,000,019`

A palindrome’s structure makes this a modular counting problem, not an enumeration problem.

## 1) Linearizing a palindrome modulo `MOD`

Index decimal digits from the **least significant** end: `0 .. L-1`.

For a length-`L` palindrome, digit `d[i]` must equal digit `d[L-1-i]`.

Let `k = ceil(L/2)`. Then the palindrome is determined by `k` digits:

- `d[0]` is both the last and first digit, so **it must be nonzero** (`1..9`) to ensure the number has exactly `L` digits.
- other digits are `0..9`
- if `L` is odd, the middle digit appears only once

Modulo `MOD`, the value is a **linear combination** of the independent digits:

- even `L`:
  - weight `w[i] = 10^i + 10^(L-1-i) (mod MOD)`
- odd `L`:
  - same weights except the middle digit uses `w[mid] = 10^mid (mod MOD)`

So for fixed `L`, the problem becomes:

> Count digit assignments with `d[0] in 1..9`, others `0..9`, such that  
> `sum(d[i] * w[i]) ≡ 0 (mod MOD)`.

## 2) Meet-in-the-middle for `k <= 14`

For `L <= 28`, `k <= 14`.

Split the `k` digits into two parts of size at most `7` digits each.

- Enumerate all sums of the left half and store frequencies.
- Enumerate all sums of the right half and look up the matching complement modulo `MOD`.

With at most `10^7` combinations on one side, this is feasible in pure Python.

## 3) Bit-sliced pair-sums for `k >= 15` (lengths 29–32)

For lengths 29–32, `k` is 15 or 16, and a naïve meet-in-the-middle would require ~`10^8` iterations, which is slow in Python.

Instead:

1. Split the digits into **4-digit blocks** (sizes like `4,4,4,3/4`).
2. Compute the distribution of **pair sums of two blocks** without enumerating all `10^8` pairs:
   - represent the multiset of residues of one block as a small number of **bitplanes**
   - for each residue in the other block, **rotate** those bitplanes by that residue (cyclic shift modulo `MOD`)
   - add the rotated planes into an accumulator using bitwise XOR/AND with carry (“bit-sliced addition”)

This uses Python big integers as dense bitsets (about `MOD` bits ≈ 1.25 MB per plane) and leverages fast C-level bit operations.

Finally, the count for a length is the dot product of the “left pair-sum” distribution with the “right pair-sum” distribution (built as *negated* sums so indices align), computed by:

- `sum_{i,j} 2^(i+j) * popcount(planeA[i] & planeB[j])`

## 4) Statement checks

`main.py` includes brute-force asserts for the statement’s small examples (divisor 109):

- smallest palindromes divisible by 109 are `545, 5995, 15151`
- there are `9` positive palindromes `< 100000` divisible by 109

These confirm the interpretation of “positive palindromes” (excluding 0).
