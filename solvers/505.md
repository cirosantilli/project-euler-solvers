# Project Euler 505 ‚ÄĒ Bidirectional Recurrence

This repository contains a fast (derived) solution for **Project Euler 505** without any hard-coding.

## Problem objects

Let `M = 2^60` and `C = M-1`.

**Sequence `x(k)`** (modulo `M`):

- `x(0)=0, x(1)=1`
- for `k ‚Č• 1`:
  - `x(2k)   = (3¬∑x(k) + 2¬∑x(‚Ćäk/2‚Ćč)) mod M`
  - `x(2k+1) = (2¬∑x(k) + 3¬∑x(‚Ćäk/2‚Ćč)) mod M`

**Truncated tree `y_n(k)`**:

- `y_n(k) = x(k)` if `k ‚Č• n`
- `y_n(k) = C ‚ąí max(y_n(2k), y_n(2k+1))` if `k < n`

Finally:

- `A(n) = y_n(1)`

The statement gives:
- `A(4)=8`
- `A(10)=2^60‚ąí34`
- `A(1000)=101881`

These are asserted in `main.py`.

## Main techniques used

### 1) Turn ‚Äúcomplement of max‚ÄĚ into a standard minimax

Define a depth-dependent transform on node values:

- if `depth(k)` is even: `s_n(k) = y_n(k)`
- if `depth(k)` is odd:  `s_n(k) = C ‚ąí y_n(k)`

(Here `depth(k)=‚Ćälog2(k)‚Ćč` in the binary heap.)

Using `y_n(k) = C ‚ąí max(y_n(2k), y_n(2k+1))` and the fact that `C‚ąí¬∑` reverses comparisons, you get:

- if `k < n` and `depth(k)` is even: `s_n(k) = min(s_n(2k), s_n(2k+1))`
- if `k < n` and `depth(k)` is odd:  `s_n(k) = max(s_n(2k), s_n(2k+1))`

So the recursion becomes an **ordinary minimax** on a binary tree with alternating `min/max` by depth.

At the root, `depth(1)=0` is even, so:

> `A(n) = y_n(1) = s_n(1)`.

---

### 2) Compute `x(k)` in `O(log k)` from the bits of `k`

Define the state vector:

`v(k) = [ x(k), x(‚Ćäk/2‚Ćč) ]^T`

Then the recurrence gives two linear updates:

- `v(2k)   = M0 ¬∑ v(k)`, where `M0 = [[3,2],[1,0]]`
- `v(2k+1) = M1 ¬∑ v(k)`, where `M1 = [[2,3],[1,0]]`

Starting from `v(1) = [1,0]^T`, walking the binary digits of `k` (excluding the leading `1`) and applying `M0`/`M1` computes `x(k)` in **O(log k)** time.

This is used whenever the minimax needs the payoff at a specific leaf.

---

### 3) Perfect subtrees have a deterministic minimax winner

A key monotonicity fact:

- For any state with `x(k) ‚Č• x(‚Ćäk/2‚Ćč)`, applying `M0` yields a state whose `x`-component is **‚Č•** the one from applying `M1`.
- As a result, within any perfect subtree, flipping a decision bit from `0` to `1` never increases the reachable `x`.

This implies in a **perfect** subtree of depth `D`, the minimax winner leaf depends **only** on:
- the subtree depth `D`
- whether the subtree root is a `min` or a `max`
- whether the leaf payoff is `x(leaf)` or `C‚ąíx(leaf)` (which simply reverses comparisons)

For payoff `x(leaf)` and a **MIN** at the subtree root, the winner offset inside the interval is:

- if `D` even: `(2^(D+1) ‚ąí 2)/3`  (base‚ÄĎ4: `22‚Ä¶2`)
- if `D` odd:  `(2^(D+1) ‚ąí 1)/3`  (base‚ÄĎ4: `11‚Ä¶1`)

If the subtree root is **MAX**, the winner is the MIN-pattern with one fewer level.

This turns ‚Äúevaluate a whole perfect subtree‚ÄĚ into **O(1)** (plus one `x(k)` call).

---

### 4) ‚ÄúUniformize‚ÄĚ the truncated tree so the boundary is a single cut

Let `d = ‚Ćälog2(n)‚Ćč` and `m = 2^(d+1)`. Then `n ‚ąą [m/2, m)`.

All frontier leaves of the original truncated tree lie at depths `d` or `d+1`.  
We can convert it to a **complete** tree with all leaves at depth `d+1`:

- The complete leaf index range is `j ‚ąą [m, 2m)`.
- There is a single breakpoint at `j = 2n`.

In `s`-space, leaf payoffs are:

- For `j < 2n` (region A): either `x(j)` or `C‚ąíx(j)` (depending on parity of `d+1`)
- For `j ‚Č• 2n` (region B): payoffs depend only on `u=j//2`, i.e. either `x(u)` or `C‚ąíx(u)`

Crucially, **only one boundary point** (`2n`) separates the two uniform regimes.

---

### 5) Recurse only on intervals that straddle the boundary

Now the minimax is on a perfect tree over the interval `[m, 2m)`.

If a node interval lies entirely in region A or region B, we evaluate it in **O(1)** using the deterministic winner formulas (and `x(k)`).

Only intervals that **cross** the single boundary `2n` need splitting. A single boundary can straddle at most **O(log m)** nodes in the recursion tree.

So the whole evaluation costs:
- `O(log n)` recursive nodes
- each with `O(log n)` for the few `x(k)` evaluations

In practice it‚Äôs extremely fast (a few thousand integer operations).
