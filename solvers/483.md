# Project Euler 483 — Repeated Permutation (Solution Summary)

This project computes:

- `f(P)`: the *order* of a permutation `P` (the smallest `m > 0` such that `P^m` is identity).
- `g(n)`: the average value of `f(P)^2` over all `n!` permutations of `{1..n}`.

The final program prints `g(350)` in scientific notation rounded to **10 significant digits**, and verifies the example fractions in the statement using exact arithmetic.

---

## Key Mathematical Ideas

### 1) Permutations as Cycle Decompositions
Every permutation decomposes uniquely into disjoint cycles.
If cycle lengths are:

\[
c_1, c_2, \dots, c_r
\]

then the order is:

\[
f(P) = \mathrm{lcm}(c_1, c_2, \dots, c_r)
\]

So we need the expected value of:

\[
\mathrm{lcm}(c_1, \dots, c_r)^2
\]

---

### 2) Cycle Index / Exponential Generating Function
For permutations, the standard cycle counting fact is:

- The contribution of having `a` cycles of length `k` is:

\[
\frac{1}{k^a a!}
\]

The full exponential generating function is:

\[
\prod_{k \ge 1} \exp(x^k / k)
\]

This exactly enumerates permutations by cycle structure.

---

## Main Algorithm (DP over Size and LCM)

We build a dynamic programming table where:

- `dp[s][L]` = total weight of cycle selections using `s` elements whose LCM is `L`

Then we update cycle length by cycle length using the series:

\[
\exp(x^k/k)=\sum_{a\ge0} \frac{x^{ak}}{k^a a!}
\]

Finally the answer is:

\[
g(n)=\sum_L dp[n][L]\cdot L^2
\]

---

## Performance Technique: Bitmask Encoding of LCM

Direct LCM with big integers is expensive, so we represent each LCM as a **bitmask** encoding prime-power exponents.

For each prime \(p\), reserve one bit per exponent:

- exponent `e` means the lowest `e` bits in that prime’s block are set.

Example: exponent 3 ⇒ bits `111`

Then:

- `LCM(maskA, maskB) = maskA OR maskB`

So LCM updates become **fast bitwise OR operations**.

At the end, masks are decoded back into integers to compute \(L^2\).
