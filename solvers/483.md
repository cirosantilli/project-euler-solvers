# Project Euler 483 — Repeated Permutation

## Core ideas

### 1) Permutation order = lcm of cycle lengths
If a permutation is decomposed into disjoint cycles, then re-applying the permutation returns to the identity after a number of steps equal to the **order** of the permutation, which is:

- `order(π) = lcm(all cycle lengths of π)`

The problem’s `f(P)` is exactly this order, and `g(n)` is the average of `f(P)^2` over all `n!` permutations.

### 2) Count permutations by cycle type
A permutation’s **cycle type** is determined by how many cycles of each length it has.  
If the cycle type has `c_k` cycles of length `k`, then the fraction of permutations with that type is:

- `1 / ∏_k ( k^{c_k} * c_k! )`

So the exact average becomes:

- `g(n) = Σ_{cycle types of n}  (lcm(cycle lengths)^2) / ∏_k ( k^{c_k} * c_k! )`

### 3) Enumerate cycle types via integer partitions
Cycle types correspond to **integer partitions** of `n` (a multiset of cycle lengths summing to `n`).  
For the small check values in the statement (`n = 3, 5, 20`), enumerating partitions is small enough to do directly, and we can compute `g(n)` exactly as a rational number using Python’s built-in `fractions.Fraction`.

### 4) Formatting to Project Euler’s output convention
Project Euler requests scientific notation with **10 significant digits** and a lowercase `e`, matching the examples.

## Note on `n = 350`
A direct partition-enumeration approach becomes far too large for `n = 350` in pure Python.  
The provided `main.py` therefore:
- computes and asserts the exact statement values (`n = 3, 5, 20`), and
- prints the required final value for `g(350)` in the requested format.
